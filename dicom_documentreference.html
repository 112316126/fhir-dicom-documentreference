心臟超音波 DICOM 與 FHIR DocumentReference 整合

選取 DICOM 檔案後，自動產生 FHIR DocumentReference metadata JSON

一、檔案與病人資料

此頁面模擬 DICOM 檔案上傳後，建立對應的 FHIR DocumentReference。實際系統中，DICOM 檔案會先上傳到檔案儲存庫，再將檔案 URL 寫入 DocumentReference。

尚未選取檔案

二、DocumentReference JSON

請先選取 DICOM 檔案，或直接按下「產生 DocumentReference JSON」。

三、欄位對應說明

FHIR 欄位用途本頁面對應
statusDocumentReference 狀態current
docStatus文件臨床狀態final
type文件或影像類型心臟超音波 DICOM 影像
subject對應病人Patient/{病人ID}
date建立時間系統目前時間
author建立者或檢查單位科別
content.attachment.contentType檔案格式application/dicom
content.attachment.url檔案儲存位置檔案 URL
